Architecture decision: tree-sitter as the core parser (not LSP). Universal code model with pure-function metrics. Per-language .scm query files as adapters — adding a language requires only writing one .scm file + plugin.toml manifest, no engine code changes. LSP deferred to V2 for cross-file analysis. Language plugins distributed as .so/.dylib/.dll, top 10 grammars compiled in, others downloaded on demand with checksum pinning. Three operating modes (guidance/tiered/gate) allow teams to adopt incrementally. Code roles (app/library/test/script/generated/vendor) are first-class — test code has stricter cyclomatic/nesting/cognitive limits because tests with branches are smelly.
